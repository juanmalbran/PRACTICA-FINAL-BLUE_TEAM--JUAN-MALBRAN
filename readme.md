# Nota sobre el direccionamiento IP

Durante el desarrollo de esta práctica se realizaron ajustes y reconstrucciones parciales de la infraestructura virtual debido a la necesidad de recrear determinados entornos y máquinas utilizados originalmente. Como consecuencia, algunas capturas o referencias puntuales pueden mostrar un esquema de direccionamiento anterior (por ejemplo: 10.1, 20.1, 20.99 o 25.1), mientras que la configuración final documentada utiliza las redes 192.168.100.0/24, 192.168.200.0/24 y 192.168.250.0/24.

Estas diferencias afectan únicamente a la numeración empleada en determinadas fases de la práctica y no modifican la arquitectura, segmentación ni el funcionamiento descrito en el proyecto. En todos los casos se mantiene la misma distribución lógica de redes (LAN, DMZ y DMZ2), los mismos servicios desplegados, las mismas reglas de seguridad y el mismo flujo de monitorización hacia Elastic Cloud.

Por tanto, cualquier referencia a esquemas de direccionamiento anteriores debe interpretarse como equivalente funcional a la configuración final presentada en este documento.
